# Agent Guide: `study` 카테고리

이 문서는 `category: study` 글을 작성하는 에이전트를 위한 가이드다.
반드시 [style.md](../style.md)의 GLOBAL RULES를 먼저 따르고, 이 문서의 규칙을 추가로 적용한다.

---

## 1. 이 카테고리가 담는 것

`study`는 **직접 배운 것을 기록하는 글**이다. 수업, 스터디, 자격증 준비, 혼자 책이나 강의를 보며 공부한 내용을 담는다.
정보를 나열하는 것이 아니라, 배우는 사람의 시각에서 왜 어려웠는지, 어디서 막혔는지, 어떻게 이해했는지를 함께 적는다.

---

## 2. 서브카테고리별 성격

### `bootcamp` — 부트캠프 수업 일별 기록

- 하루 단위로 배운 내용을 정리하는 시리즈 글이다.
- 시리즈명과 회차 번호를 frontmatter의 `series`, `seriesOrder`에 명시한다.
- 타이틀 형식: `[주제] N일차: 소제목` 또는 `[주제] DayN: 소제목`
- 예: `Python 기초 1일차: 변수, 자료형, 출력`

### `certification` — 자격증 스터디 기록

- 시험 준비 단계별 정리 글이다. 스터디 그룹 운영 방식, 발표 분담, 개인 복습 포인트를 포함한다.
- 타이틀 형식: `[자격증 이름] [N회차] 소제목`
- 예: `[AWS CLF-C02] [1회차] 클라우드 컴퓨팅`
- 글 말미에 **시험 전 복습 포인트** 섹션을 두면 좋다.
- 발표 자료 다운로드 링크를 제공하는 경우 본문 마지막에 배치한다.

### `self-study` — 자기 주도 학습 기록

- 혼자 책이나 강의를 보며 공부한 내용이다. 스팸 분류 프로젝트처럼 특정 주제를 깊이 파는 시리즈도 포함된다.
- 실험 과정과 결과, 막힌 지점과 해결 방식을 서술형으로 담는다.

## 3. 문체 및 구조

- 언어는 소스 메모 언어 기준으로 결정한다 (한국어 메모 → 한국어 글).
- 글의 시작은 **"왜 이것을 배웠는가" 또는 "오늘 무엇을 했는가"**로 자연스럽게 연다.
- 코드 블록은 반드시 언어를 명시한다 (`\`\`\`python`, `\`\`\`java` 등).
- 헷갈렸던 부분, 실수한 포인트를 솔직하게 적는다. "이건 쉬웠다"만 있으면 안 된다.
- 복습 Q&A를 `<details>` / `<summary>` 태그로 넣으면 좋다 (bootcamp 글에 자주 사용됨).

### 권장 섹션 순서 (bootcamp / certification)

1. **오늘 다룬 범위** — 짧게, 무엇을 배웠는지 목록 또는 한 문단
2. **핵심 개념 정리** — 각 개념을 서술형으로, 코드 예시 포함
3. **헷갈렸던 점 / 실수 포인트** — 솔직하게
4. **복습 Q&A** (선택, `<details>` 사용)
5. **한 줄 정리 또는 다음 회차 예고**

### 권장 섹션 순서 (certification)

1. **이번 회차에서 다룬 내용** — 스터디 운영 방식, 발표자 배분 포함
2. **왜 이 개념이 중요한가** — 맥락 설명
3. **개념별 상세 정리** — 비교표, 판단 기준 포함
4. **이번 회차에서 남겨둘 포인트** — 여전히 헷갈리는 것, 발표를 준비하면서 느낀 점
5. **시험 전에 다시 볼 포인트** — 핵심 요약
6. **자료 다운로드** (있는 경우)

## 4. Frontmatter 작성법

```yaml
---
title: "Python 기초 1일차: 변수, 자료형, 출력"
slug: python-fundamentals-day1-variables-types-and-output
date: 2025-12-01
author: Evan Yoon
category: study
subcategory: bootcamp          # bootcamp | certification | self-study
description: "오늘 배운 내용을 한 문장으로 요약."
thumbnail: ""                  # 이미지 있으면 /images/posts/시리즈명/cover.png
tags:
  - python
  - study
  - 관련-기술-태그
readTime: 6
series: "Python Self-Study"    # 시리즈명 (단발 글이면 "" 비워도 됨)
seriesOrder: 1
featured: false
draft: false
toc: true
---
```

**주의사항**
- `subcategory`는 반드시 `bootcamp`, `certification`, `self-study` 중 하나.
- `series`와 `seriesOrder`는 시리즈 글에는 필수, 단발 글은 `series: ""`.
- `toc: true`는 중간 소제목이 3개 이상인 경우 사용.

## 5. 실제 글 패턴 예시

### bootcamp 글 도입부 패턴

```
12월 첫 평일에는 Python을 정말 기초부터 다시 정리했다. 이미 한 번 본 내용이라고
생각했지만, 막상 코드를 직접 쓰려니 숫자와 문자열을 다루는 감각이 아직 완전히
손에 익지는 않았기 때문이다. 그래서 오늘은 욕심내지 않고 변수, 자료형, 출력처럼
가장 기본적인 부분만 다시 확인했다.
```

→ "왜 다시 이것을 보는가"를 솔직하게 써서 자연스럽게 시작한다.

### certification 글 도입부 패턴

```
2026년 1월 6일 화요일, AWS Certified Cloud Practitioner(CLF-C02) 스터디 1회차를
진행했다. 첫 회차 주제는 가장 기초가 되는 클라우드 컴퓨팅 기본 개념이었다.

이날 다룬 범위는 네 가지였다.
1. 클라우드 컴퓨팅의 정의와 온프레미스 비교
2. ...
```

→ 날짜, 스터디 회차, 이날 범위를 명시한 후 본문으로 진입.

## 6. 하면 안 되는 것

- 개념을 나열만 하고 "이것은 이렇다"식으로만 설명하기 (배우는 사람의 목소리가 없어짐)
- 코드 블록 없이 기술 개념만 말로 설명하기
- "이번 포스팅에서는 X를 알아보겠습니다" 같은 블로그 인트로 클리셰
- 결론을 억지로 감동적으로 마무리하기 ("오늘도 열심히 공부했습니다!")
- AI가 요약한 것처럼 읽히는 bullet 나열형 정리
