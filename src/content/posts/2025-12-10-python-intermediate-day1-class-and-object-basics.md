## 중급 파트의 시작은 클래스였다

기초 파트에서는 값과 흐름을 다루는 방법을 배웠다면, 중급 파트에 들어오자마자 클래스라는 개념이 나왔다. 처음엔 갑자기 어렵게 느껴졌지만, 실습 코드를 다시 보니 클래스는 결국 "관련 있는 데이터와 기능을 묶는 설계도"로 이해하면 조금 낫다.

오늘은 클래스와 객체의 차이를 분명하게 구분하는 데 집중했다.

## 클래스는 설계도, 객체는 실제 결과물

수업 시간 실습에서는 자동차 예제를 통해 클래스를 만들었다.

```python
class Car:
    def init(self):
        self.color = "black"
        self.speed = 100

    def info(self):
        print(f"자동차 색상은 {self.color}, 속도는 {self.speed}")
```

이 코드는 `Car`라는 틀을 만드는 과정이다. 그리고 실제로 값을 가지는 대상을 만들려면 객체를 생성해야 한다.

```python
redcar = Car()
redcar.init()
redcar.color = "red"
redcar.speed = 150
redcar.info()
```

이 부분을 다시 보면서, 클래스는 개념이고 객체는 실제 사용 대상이라는 점이 조금 더 명확해졌다.

## 속성과 메서드는 함께 움직인다

클래스를 배우면서 새롭게 들어온 용어가 속성과 메서드였다.

- 속성: 객체가 가지고 있는 값
- 메서드: 객체가 수행하는 기능

위 자동차 예제로 보면 `color`, `speed`는 속성이고 `info()`는 메서드다. 기초 단계에서는 변수와 함수가 따로 있었다면, 클래스에서는 이 둘이 하나의 대상 안에 묶인다.

이 구조가 중요한 이유는 데이터와 기능이 서로 연결되기 때문이다. 자동차의 색상과 속도를 따로따로 흩어놓는 대신, 자동차라는 단위로 함께 관리할 수 있다.

## self가 왜 필요한지도 조금 보였다

오늘 가장 낯설었던 건 `self`였다. 메서드 정의에서 항상 첫 번째 인자로 들어가고, 속성에 접근할 때도 `self.color`처럼 써야 했다.

처음에는 문법 규칙처럼 외워야 하나 싶었는데, 다시 정리해보니 `self`는 "지금 이 객체 자신의 값"을 가리키는 표시라고 이해하면 된다. 즉, 클래스 내부에서 각 객체의 상태를 구분하는 기준점 역할을 한다.

## 오늘 헷갈렸던 점

클래스와 객체가 가장 헷갈렸다. `Car`는 클래스 이름이고, `redcar`는 실제 객체인데 초반에는 둘을 같은 의미처럼 읽게 된다.

또 `self`는 자동으로 붙는 것이 아니라 메서드 정의에 직접 적어야 한다는 점도 계속 신경 써야 했다. Python이 객체지향을 어떻게 표현하는지 처음 맛본 날이었다.

## 오늘의 실수 포인트

- 클래스 자체와 객체를 구분하지 않으면 구조가 흐려진다.
- 속성은 `self.이름` 형태로 다룬다.
- 메서드는 클래스 안에 있지만, 실제로는 객체를 통해 호출한다.
