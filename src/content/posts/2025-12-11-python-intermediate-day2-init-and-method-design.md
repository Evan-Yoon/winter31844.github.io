## 오늘은 객체가 처음 만들어지는 순간을 봤다

중급 2일차에는 `__init__` 생성자를 배웠다. 어제는 객체를 만든 뒤 따로 `init()` 같은 메서드를 호출해 속성을 설정했다면, 오늘은 객체가 만들어질 때 필요한 값을 한 번에 넣는 방식으로 넘어갔다.

이 차이가 생각보다 컸다. 코드가 더 자연스럽고, 객체의 초기 상태를 더 명확하게 정할 수 있기 때문이다.

## __init__은 객체 생성 시 자동 실행된다

실습 코드에서는 아래처럼 생성자를 사용하는 클래스를 만들었다.

```python
class CarI:
    def __init__(self, c, s):
        self.color = c
        self.speed = s

    def info(self):
        return f"자동차 색상은 {self.color}, 속도는 {self.speed}"
```

이제 객체를 만들 때 바로 값을 넣을 수 있다.

```python
car_red = CarI("Red", 80)
print(car_red.info())
```

이 구조를 보면서 `__init__`은 객체의 출발 상태를 정해주는 역할이라고 이해했다. 나중에 어떤 클래스든 처음 만들어질 때 필요한 정보가 있을 텐데, 그걸 이곳에 모아두는 것이다.

## 메서드는 객체가 할 수 있는 행동이다

생성자와 함께 다시 본 건 메서드의 역할이었다. 메서드는 객체의 속성을 활용해서 특정 결과를 만들어낸다. 예제에서는 `info()`가 그 역할을 했다.

기초 단계에서 함수가 독립적으로 존재했다면, 클래스 안의 메서드는 객체의 속성과 연결된 함수라고 보면 이해가 편했다.

```python
class Book:
    def __init__(self, title, author):
        self.title = title
        self.author = author

    def get_title(self):
        return self.title

    def get_author(self):
        return self.author
```

이 코드는 객체가 어떤 값을 갖고 있으며, 그 값을 어떤 방식으로 제공할지 함께 정의하고 있다.

## 오늘 가장 중요했던 감각

오늘 정리하면서 느낀 핵심은 "객체를 만들 때 기본값과 필요한 값을 어떻게 설계할 것인가"였다. 생성자를 쓰면 나중에 객체가 많아져도 초기 상태를 일정하게 맞출 수 있다.

예를 들어 자동차 객체를 만들 때 색상과 속도를 빠뜨리지 않게 만들 수 있고, 책 객체를 만들 때 제목과 저자를 반드시 넣도록 강제할 수도 있다. 이건 단순 문법보다 설계에 가까운 문제라서, 중급으로 넘어왔다는 느낌이 들었다.

## 오늘 헷갈렸던 점

`__init__` 이름은 특수해서 철자를 정확히 써야 한다는 점이 계속 신경 쓰였다. 언더바 개수 하나만 틀려도 작동하지 않는다.

또 생성자 안에서 받는 매개변수 이름과, 실제 객체 속성 이름이 어떻게 연결되는지도 처음에는 조금 어색했다. 하지만 `self.color = c`처럼 읽어보니 "받은 값을 객체 속성에 저장한다"는 의미로 이해됐다.

## 오늘의 실수 포인트

- `__init__` 철자를 틀리면 생성자가 실행되지 않는다.
- 생성자 매개변수와 객체 속성은 다를 수 있지만 연결 관계를 분명히 써야 한다.
- 메서드는 객체의 속성을 활용하는 기능이라는 점을 잊지 않아야 한다.

3. 객체 생성 시 필요한 값을 생성자로 받는 이유는 무엇인가?
