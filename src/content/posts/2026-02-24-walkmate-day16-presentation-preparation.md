## 📂 발표 자료 구조: "공부 많이 된" 기록들

PPT는 총 5가지의 핵심 챕터로 구성하여, 우리가 단순한 기능 구현을 넘어 기술적 깊이를 고민했음을 강조함.

### 1. 문제 인식과 Walkmate의 탄생
시각장애인과 보행 약자들이 겪는 실제적인 이동의 어려움을 데이터로 제시함. 기존 내비게이션이 해결하지 못한 '세밀한 보행 안내'의 필요성을 역설함.

### 2. 핵심 기술 스택 (Tech Stack)
- **Frontend**: Capacitor, React, Tailwind CSS
- **AI/Vision**: TFLite, Custom Object Detection Models (Hazard Recognition)
- **Maps**: Tmap API Integration
- **DevOps**: GitHub Actions, Git Flow

### 3. 주요 기능 및 시연
- **실시간 위험 요소 감지**: VisionCamera를 통한 턱, 볼라드, 계단 인식.
- **스마트 가이딩 UI**: 보행자 중심의 직관적인 방향 지시.
- **관리자 히트맵**: 사고 다발 지역 및 위험 요소 밀집도 시각화.

### 4. 트러블슈팅: 위기를 기회로
가장 공들여 작성한 부분임. 프로젝트 도중 겪었던 **Git 브랜치 동기화 문제**와 **저속 보행 시 방향 인식 불안정** 문제를 어떻게 센서 퓨전과 필터링 알고리즘으로 해결하려 했는지 상세히 담음.

### 5. 성찰 및 향후 계획
팀명처럼 이번 프로젝트를 통해 각 팀원이 얻은 기술적 성취를 정리함. "공부가 많이 된다"는 말이 단순한 농담이 아니라, 실질적인 역량 강화로 이어졌음을 확인함.

## 💭 Reflections

### 기술의 무게
40여 장의 슬라이드를 채우며 느낀 점은, 우리가 만든 코드 한 줄 한 줄에 사용자들의 안전이 달려있다는 책임감이었음. PPT를 만들며 기술적 결함을 다시 검토하고 보완할 점을 찾는 과정 자체가 또 하나의 개발 단계였음.

### 협업의 완성
혼자라면 절대 만들 수 없었을 결과물임. 서로의 코드를 리뷰하고, 밤새워 버그를 잡고, 디자인을 다듬던 시간들이 슬라이드 곳곳에 녹아 있음. 발표 자료는 그 치열했던 16일의 기록임.
