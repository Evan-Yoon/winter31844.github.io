## 오늘의 Mongle: 한 걸음 더 스마트하게

오늘은 유난히 결과물이 눈에 띄는 하루였다. 결혼 준비라는 복잡한 여정을 조금이라도 더 쉽고 똑똑하게 만들어주기 위해 고민했던 기능들이 하나둘씩 제 자리를 찾아갔다. 특히 **AI 예산 관리**와 **RAG 기반 챗봇**은 우리 팀이 가장 공을 들이고 있는 핵심 기능들이다.

### 💰 AI로 완성하는 스마트한 예산 관리

웨딩 준비에서 가장 민감한 부분은 역시 '돈'이다. 하지만 단순히 숫자를 기록하는 것만으로는 부족하다. 이번에 추가된 **AI 예산 최적화** 기능은 현재의 예산 사용 패턴을 분석하고, 남은 비용을 어떻게 배분하는 것이 효율적인지 가이드를 제공한다.

- **예산 관리 AI 버그 수정**: 연동 과정에서 발생했던 소소한 데이터 매핑 오류들을 잡아냈다.
- **AI 분석 기능 추가**: 이제 사용자의 상황에 맞는 실질적인 분석 결과가 타임라인과 예산 탭에서 실시간으로 제공된다.
- **UI 최적화**: 예산 최적화 모달과 상담 신청 모달 등 사용자가 직접적으로 AI의 비서를 경험할 수 있는 인터페이스를 강화했다.

### 🤖 더욱 똑똑해진 RAG 기반 챗봇

단순히 사전에 정의된 답변만 내놓는 챗봇은 웨딩 플래너의 역할을 대신할 수 없다. 우리는 **RAG(Retrieval-Augmented Generation)** 기능을 도입하여, 우리 플랫폼이 가진 방대한 업체 데이터와 노하우를 기반으로 챗봇이 훨씬 더 정확하고 구체적인 정보를 제공할 수 있도록 고도화했다.

### 🏗️ 내실을 다지는 과정

기능 추가만큼 중요한 것이 안정적인 구조다.

- **Supabase 스키마 정렬**: 프론트엔드의 페이로드와 백엔드 모델을 Supabase 스키마와 완벽하게 일치시켜 데이터 무결성을 확보했다.
- **사용자 구분(플래너/일반회원)**: 로그인과 가입 과정에서 사용자의 역할을 명확히 구분하여, 각 역할에 맞는 최적화된 경험을 제공할 준비를 마쳤다.
- **코드 리팩토링**: index와 id 최적화를 통해 전반적인 앱 성능과 데이터 접근 속도를 개선했다.

## 앞으로의 Mongle

오늘의 개발은 단순히 기능을 더한 것을 넘어, **"AI가 실제로 사용자에게 어떤 도움을 줄 수 있는가"**에 대한 답변을 찾아가는 과정이었다. 아직은 다듬어야 할 부분들이 많지만, 사용자가 우리 앱을 켰을 때 "정말 나를 잘 아는 전문가와 함께하고 있다"는 느낌을 받을 수 있도록 디테일을 더해갈 예정이다.

내일은 오늘 구현한 RAG 기반 챗봇의 성능을 더욱 정교하게 튜닝하고, 사용자 테스트를 통해 발견된 마이너한 UI 이슈들을 해결하는 데 집중할 계획이다. 한 걸음씩 완성되어가는 Mongle을 보며 뿌듯함을 느낀다.
