## 1. 프로젝트 개요

### 1.1 프로젝트 주제

```
YOLOv11n과 TFLite를 활용하여 시각장애인에게 실시간 보행 장애물 정보를 제공하고, 인프라 문제를 자동으로 신고하는 온디바이스 AI 서비스
```

---

### 1.2 선정 배경 및 기획 의도

### (1) 문제 인식 (What)
- 현행 보행 보조 수단(지팡이, 안내견)은 돌발 장애물(무단 방치 킥보드, 볼라드 등) 대응에 한계가 있음.
- 훼손된 점자블록이나 포트홀 등 인프라 결함이 방치되어 2차 사고 위험이 높음.
- 기존 AI 보행 보조 앱은 서버 통신 지연으로 인해 실시간성이 떨어짐.

### (2) 문제 발생 원인 (Why)
- 도시 환경의 급격한 변화(개인용 이동장치 보급)로 인한 새로운 장애물 등장.
- 인프라 관리 기관의 인력 부족으로 인한 실시간 모니터링 공백.
- 딥러닝 모델의 무거운 용량과 연산량으로 인한 모바일 환경 최적화의 어려움.

### (3) 해결 방향 (How)
- **온디바이스 AI**: YOLOv11 Nano 모델을 경량화(6MB 이하)하여 지연 시간 없는 실시간 객체 탐지 구현.
- **자동 민원 신고**: 장애물 탐지 시 위치 정보와 함께 행정안전부 등 관련 기관에 자동으로 리포트 생성.
- **다중 피드백**: 골전도 이어폰을 통한 음성 안내 및 고대비 HUD UI를 통한 저시력자 시각 보조.

---

### 1.3 프로젝트 특화 포인트 및 차별성

| 구분 | 기존 서비스 (예: PathPal) | 본 프로젝트 (Walkmate) |
| --- | --- | --- |
| 기술 방식 | 서버 통신 기반 (Latency 있음) | **온디바이스 AI (실시간성 확보)** |
| 핵심 가치 | 단순 길 안내 및 장애물 알림 | **장애물 탐지 + 자동 민원 신고 결합** |
| 사용자 경험 | 일반적인 스마트폰 UI | **골전도 피드백 + 고대비 HUD UI** |
| 모델 성능 | 기존 무거운 모델 사용 | **YOLOv11n + TFLite 경량화 (3MB 내외)** |

## 2. 핵심 기능 정의 (MVP)

### 2.1 핵심 기능 목록

| 기능명 | 기능 설명 | 우선순위 |
| --- | --- | --- |
| **실시간 객체 탐지** | 점자블록(훼손 포함), 킥보드, 볼라드, 포트홀 탐지 | **Must / P1** |
| **음성/진동 피드백** | 골전도 이어폰을 활용한 위험 방향 및 거리 안내 | **Must / P1** |
| **자동 신고 시스템** | 탐지된 인프라 결함을 위치 정보와 결합하여 자동 리포트 | **Must / P1** |
| **고대비 HUD UI** | 저시력자를 위한 고대비 스타일의 메인 화면 | **Should / P2** |
| **데이터 증강 학습** | 다양한 도로 상황 데이터셋(AI Hub) 기반 전이 학습 | **Should / P2** |

## 3. 기술 스택

| 구분 | 기술 | 활용 목적 |
| --- | --- | --- |

## 4. 기대 효과

- **사회적 가치**: 시각장애인의 보행 안전권 확보 및 도시 인프라 개선 기여.
- **기술적 성과**: 최신 YOLOv11 모델의 모바일 최적화 및 실무 적용 능력 배양.
- **사용자 편의**: 실시간성 확보를 통한 보행 중 사고 예방 극대화.
