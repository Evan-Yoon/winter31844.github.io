Instance Store 개념

빠르지만 휘발성인 물리적 디스크

  • EC2 인스턴스가 실행되는 물리적 호스트에 직접 연결
  • Ephemeral Storage

Instance Store 특징

  • 인스턴스 중지/종료 시 데이터 삭제
  • 네트워크 미사용 → 매우 높은 IOPS
  • 캐시, 버퍼, 스크래치 데이터

Instance Store 구조 이해

Instance Store는 EC2가 실행 중인 물리 호스트에 종속된 디스크입니다.

각 EC2 인스턴스는 동일 호스트 내 Instance Store만 사용 가능
  • ✔️ 다른 호스트의 인스턴스와 공유 불가
  • ✔️ 인스턴스 종료/중지 시 데이터 소실
  • ✔️ 네트워크 미사용 → 초고속 I/O

Amazon EFS 개념

여러 서버가 공유하는 네트워크 파일 시스템

  • 완전 관리형 NFS
  • Linux 기반 EC2 전용

Amazon EFS 구조 이해

EFS는 각 AZ마다 Mount Target를 두고, 여러 EC2 인스턴스가 동시에 접근합니다.

Multi-AZ 환경에서 EC2들이 하나의 EFS 파일 시스템을 공유
  • ✔️ AZ 장애 시에도 다른 AZ에서 접근 가능
  • ✔️ 웹 서버, 컨테이너 환경에 적합
  • ✔️ Linux 기반 NFS 공유 스토리지

EFS 특징 및 제약

  • Multi-AZ, 다수 EC2 동시 접근
  • 자동 확장 / 고가용성
  • Linux 전용 (Windows → FSx)
