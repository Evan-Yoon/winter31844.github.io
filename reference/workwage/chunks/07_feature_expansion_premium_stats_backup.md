# Chunk 07 — 기능 확장: 통계, 프리미엄, 백업, 알림

## 이 청크의 역할
초기 MVP에서 끝나지 않고, 실제 서비스처럼 기능이 확장된 흐름을 설명한다.

## 공개 커밋 히스토리에서 읽히는 확장 방향
초기 핵심 계산/입력 기능 이후 저장소는 다음 축으로 빠르게 확장되었다.
- 프리미엄 구독 / 페이월
- 통계 페이지
- PDF/CSV/Excel export
- 알림 설정
- Google Drive 기반 백업/복원
- 근무 패턴 자동 생성
- 병원 규정/프리셋 관련 페이지
- 법률 온보딩 페이지
- 앱 아이콘 및 UI 수정

## 왜 이 흐름이 중요하나
이 청크는 단순 학습 프로젝트가 아니라, 점점 **서비스형 앱의 구조**를 갖춰가는 과정을 보여준다.

## 블로그에 넣으면 좋은 포인트
- 처음엔 계산기였지만, 점차 “근무 기록 관리 + 리포트 + 백업 + 프리미엄 기능”을 가진 제품으로 커졌다.
- 사용 편의 기능과 수익화 기능을 함께 고민하기 시작했다.
- CSV/PDF export, 통계, 백업은 실사용자 관점에서 가치가 큰 기능이었다.

## 블로그 문장 후보
- “처음에는 계산 로직 구현이 목표였지만, 개발이 진행될수록 기록·통계·백업·구독까지 붙으며 앱의 성격이 훨씬 제품에 가까워졌다.”
- “특히 export와 백업 기능을 넣으면서 ‘내가 만드는 게 토이 프로젝트가 아니라 실제 사용 가능한 도구일 수도 있겠다’는 감각이 생겼다.”

## 블로그에서 주의할 점
- 모든 기능을 상세히 다 설명하면 글이 산만해진다.
- 핵심 서사는 “문제 해결형 MVP → 사용성 고도화 → 서비스 기능 확장” 3단 구조로 묶는 편이 좋다.
