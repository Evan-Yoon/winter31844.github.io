# Chunk 04 — UI 구상과 프롬프트 기반 제작 흐름

## 이 청크의 역할
디자인/코드 생성 도구를 이용해 어떻게 화면과 초기 구조를 빠르게 만들었는지 설명한다.

## 핵심 흐름
- UI 구상은 Stitch 프롬프트를 통해 시작했다.
- 코드 구조와 로직은 Jules 프롬프트를 통해 생성하도록 설계했다.
- 이 과정에서 중요한 것은 “예쁜 화면”을 요청하는 것이 아니라, **필요한 데이터와 계산 규칙까지 프롬프트에 명시하는 것**이었다.
- 화면은 카드형 리스트, 홈 대시보드, 입력 폼, 설정 페이지 중심으로 구성되었다.
- 프롬프트에는 리스트 뷰, 시간 선택, 공휴일 토글, 시급 입력, 상태 관리, 저장 방식 등이 구체적으로 적혔다.

## 실전에서 드러난 포인트
- 설명 칸에 단순히 “Flutter로 구현해줘”라고 쓰면 껍데기 중심 결과가 나온다.
- 반대로 로직, 입력값, 상태 갱신 방식, 저장 방식까지 적어주면 산출물의 품질이 확실히 달라진다.
- 즉, 이 프로젝트는 코드만 만든 것이 아니라 **도구에게 일을 시키는 방식도 함께 배운 과정**이었다.

## 블로그에 넣으면 좋은 포인트
- 생성형 도구를 쓸 때 “막연한 요청”과 “구조화된 요청”의 결과 차이
- UI 도구와 코드 도구의 역할 분리
- 프롬프트 자체가 일종의 기획서 역할을 했다는 점

## 블로그 문장 후보
- “이번 프로젝트에서는 코드보다 먼저, 어떤 정보를 AI에게 정확히 전달해야 원하는 결과가 나오는지부터 배웠다.”
- “디자인용 프롬프트와 구현용 프롬프트를 분리하니, 화면과 로직이 뒤섞이지 않고 훨씬 정리된 결과가 나왔다.”
