# Chunk 05 — 급여 계산 로직과 노동법 반영 포인트

## 이 청크의 역할
이 앱의 진짜 핵심인 계산 로직을 설명한다. 블로그에서 가장 기술적인 부분이 될 수 있다.

## 핵심 흐름
- 단순 시급 계산이 아니라, 다음 요소를 계산에 반영하려 했다.
  - 시작/종료 시간
  - 날짜 넘어가는 야간근무
  - 휴게 시간 차감
  - 공휴일 여부
  - 연장/야간/휴일 수당
- 특히 “20:00~05:00처럼 자정을 넘기는 근무”와 “22:00~06:00 야간 구간 겹침”이 중요한 예제로 반복해서 다뤄졌다.
- 휴게 시간은 총 체류 시간에서 차감하는 구조로 잡았다.
- 한국 노동법 기준 예시가 별도로 정리되어 있었고, 2026년 최저시급 10,320원과 야간/휴일 수당 예시가 프로젝트 설명의 중요한 근거가 되었다.

## 블로그에 넣으면 좋은 포인트
- 겉보기엔 단순한 앱처럼 보여도, 실제로는 시간 계산의 경계값 처리가 가장 어려웠다는 점
- ‘앱 개발’보다 ‘급여 규칙을 로직으로 번역하는 일’이 더 어렵게 느껴졌을 수 있다는 점
- 사용자의 HR/재무 경험이 계산 규칙 설계에 도움이 되었다는 점

## 블로그 문장 후보
- “화면을 그리는 것보다 어려웠던 건, 실제 급여 계산 규칙을 코드가 이해할 수 있는 형태로 바꾸는 일이었다.”
- “특히 자정을 넘기는 근무, 야간 시간대 겹침, 휴게 시간 차감 같은 부분은 단순 계산처럼 보여도 예외 처리가 많았다.”

## 구현 관점에서 강조할 수 있는 세부 항목
- ShiftEntry 모델 설계
- ShiftCalculator 또는 SalaryCalculator 클래스 분리
- 총 근무 시간과 실근로 시간 구분
- 휴게시간이 총근로시간보다 길 때의 validation
- 공휴일 여부를 토글로 입력받는 MVP 전략
- 이후 공공데이터 API 연동으로 확장 가능한 구조
