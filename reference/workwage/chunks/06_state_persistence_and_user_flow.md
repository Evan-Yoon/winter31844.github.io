# Chunk 06 — 상태 관리, 저장, 사용자 흐름

## 이 청크의 역할
근무를 하나 입력할 때마다 합계가 즉시 반영되고, 앱을 껐다 켜도 기록이 남아야 하는 흐름을 설명한다.

## 핵심 흐름
- 이 프로젝트에서 중요한 UX는 “입력 → 계산 → 합계 반영”이 즉시 연결되는 것이었다.
- 이를 위해 Provider / ChangeNotifier 계열 상태 관리가 논의되었다.
- 사용자가 입력한 시급과 근무 기록은 SharedPreferences 등 로컬 저장소에 남기는 방향으로 잡았다.
- 홈 화면은 단순 리스트가 아니라, **이번 달 예상 급여 / 총 근무 시간 / 누적 기록**이 한눈에 보이는 대시보드 성격을 갖게 되었다.

## 블로그에 넣으면 좋은 포인트
- 계산이 맞는 것만으로는 앱이 완성되지 않고, 사용 흐름이 자연스러워야 한다는 점
- “근무를 하나 추가하면 전체 합계가 바로 바뀌는 경험”이 앱의 핵심 피드백 루프라는 점
- 로컬 저장만으로도 MVP에서는 충분히 쓸 만한 앱 경험을 줄 수 있다는 점

## 블로그 문장 후보
- “사용자가 데이터를 입력한 뒤 결과가 바로 바뀌는 경험을 만들어야 비로소 계산기가 아니라 앱처럼 느껴졌다.”
- “초기 버전에서는 서버보다 로컬 저장이 더 중요했다. 기록이 남고, 다시 열었을 때 이어지는 경험이 MVP 완성도를 크게 좌우했다.”

## 블로그에 넣을 수 있는 배운 점
- 상태 관리가 필요한 이유를 체감했다.
- 저장소를 붙이지 않아도 로컬만으로 꽤 많은 UX를 만들 수 있다는 걸 알게 됐다.
- 입력 폼 validation과 상태 갱신 타이밍이 사용자 경험에 큰 영향을 준다는 걸 배웠다.

## 연결되는 다음 청크
- 이후 저장소 커밋 히스토리상 기능은 프리미엄/통계/백업/알림 쪽으로 크게 확장된다.
