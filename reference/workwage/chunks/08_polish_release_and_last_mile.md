# Chunk 08 — 마감 직전 폴리싱과 릴리즈 감각

## 이 청크의 역할
후반부 커밋에서 보이는 마감·수정·출시 직전 분위기를 정리한다.

## 공개 커밋 히스토리상 후반 흐름
- app icons 변경
- loading animation fix
- legal_onboarding page 수정
- UI fixing
- delete confirmation
- AndroidManifest / build.gradle.kts / 권한 / release signing 관련 추가
- last check 류의 마감 커밋

## 블로그에 넣으면 좋은 포인트
- 실제 앱 개발은 “핵심 기능 구현”으로 끝나지 않는다는 점
- 마지막에는 삭제 확인, 로딩 애니메이션, 권한, 아이콘, 온보딩 문구처럼 세부 품질 요소가 중요해진다
- 후반부에는 기능 추가보다 **다듬기와 안정화**가 더 큰 비중을 차지했다는 점

## 블로그 문장 후보
- “막상 기능이 어느 정도 갖춰지고 나니, 진짜 시간을 많이 잡아먹은 건 삭제 확인, 로딩 처리, 아이콘, 온보딩 문구 같은 마지막 10%였다.”
- “앱은 기능만으로 완성되지 않고, 마감 단계에서는 오히려 이런 디테일이 전체 완성도를 결정한다는 걸 체감했다.”

## 블로그에서 강조 가능한 학습 포인트
- 출시를 염두에 둔 Android 설정
- 권한과 서명 같은 배포 레벨 이슈
- UI 사소한 수정의 중요성
- 마감 직전 QA 감각

## 연결되는 다음 청크
- 마지막 청크에서는 이 전체 과정을 통해 느낀 점과 배운 점을 한 번에 정리한다.
