# 1차 프롬프트

아래 자료를 먼저 읽고 기억해줘.
지금 단계에서는 글을 바로 길게 쓰지 말고, **이 프로젝트의 출발점과 문제 정의, 그리고 MVP 축소 과정**을 정확히 이해하는 데 집중해줘.

## 작업 목표
- 이 앱이 왜 시작되었는지
- 왜 이 주제가 선택되었는지
- 왜 달력형 구조에서 리스트형 구조로 바뀌었는지
- 블로그에서 어떤 문제의식으로 시작하면 자연스러운지

## 출력 형식
1. 이 프로젝트의 출발 배경 5줄 요약
2. 블로그 첫머리에 쓰기 좋은 문제 정의 3가지 버전
3. “왜 달력을 버리고 리스트를 택했는가”를 설명하는 문단 초안 1개
4. 아직 부족한 정보가 있다면 질문 대신 “추가로 있으면 좋은 정보”만 짧게 적기
