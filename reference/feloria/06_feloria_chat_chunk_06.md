STEP 3 — TILESET RENDERING ENGINE

Replace placeholder rectangle rendering.

Use tileset atlas rendering.

Example:

const tileset = new Image()
tileset.src = "/assets/tilesets/overworld/terrain.png"

ctx.drawImage(
  tileset,
  tile.sx,
  tile.sy,
  TILE_SIZE,
  TILE_SIZE,
  worldX,
  worldY,
  TILE_SIZE,
  TILE_SIZE
)

Each tile ID maps to atlas coordinates.

------------------------------------------------

STEP 4 — PLAYER SPRITE

Replace the blue square player.

Load player sprite sheet from

src/assets/sprites/player

Support walking animation.

Frame layout:

row 0 → down  
row 1 → left  
row 2 → right  
row 3 → up  

Cycle animation frames during movement.

------------------------------------------------

STEP 5 — CAT SPRITES

Use cat sprite packs.

Store in:

src/assets/sprites/cats

Categories:

overworld
battle_front
battle_back

Replace placeholder battle graphics.

------------------------------------------------

STEP 6 — NPC SPRITES

Use premade-npc-spritesheets.zip.

Store in

src/assets/sprites/npcs

NPCs must support facing direction.

------------------------------------------------

STEP 7 — DEPTH SORTING

Implement Y-based rendering order.

Objects with larger Y value render later.

Allows player to walk behind trees and houses.

------------------------------------------------

STEP 8 — TALL GRASS SYSTEM

Tiles marked as tallGrass must:

trigger encounter checks  
play grass animation when player enters

STEP 12 — DEBUG RULE

If assets are missing:

Do NOT render text labels inside tiles.

Do NOT render terrain names.

Use simple colored placeholder tiles temporarily.

But keep the tileset rendering pipeline ready.

------------------------------------------------

STEP 13 — FINAL GOAL

Upgrade Feloria into a visually polished pixel RPG world.

The overworld must look like a classic monster RPG exploration map with animated sprites, detailed tiles, and layered environments instead of a debug grid.
