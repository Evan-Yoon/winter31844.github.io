## 1. 프로젝트 개요

### 1.1 프로젝트 주제

- 본 프로젝트의 주제를 한 문장으로 작성

```
AI 챗봇·일정 관리·예산 관리·업체 리스트를 통합하여 웨딩플래너와 예비부부가 실시간으로 협업할 수 있는 웨딩 플랜 AI 서비스 개발
```

---

### 1.2 선정 배경 및 기획 의도

### (1) 문제 인식 (What)

- 우리가 해결하려는 문제는 무엇인가? (3줄 이내)

```
•	결혼 준비는 수십 개의 업체 계약, 복잡한 일정, 불투명한 비용 구조가 뒤섞인 고난도 프로젝트
•	예비부부와 웨딩플래너 간 소통이 카카오톡·전화에 의존해 정보 유실 및 혼선이 잦음
•	비용 명세 공유 방식이 없어 추가 옵션·숨겨진 비용에 대한 불신이 지속적으로 발생함
```

### (2) 문제 발생 원인 (Why)

- 왜 이 문제가 발생했는가? (3가지 내외)

```
•	웨딩 준비를 위한 통합 협업 도구가 없어 여러 앱·메신저·스프레드시트를 병행 사용해야 함
•	플래너가 관리하는 정보(일정·비용·업체)가 예비부부에게 실시간으로 공유되지 않아 신뢰도 ↓
•	업체 추천이 플래너 주관에 의존하고, 실사용 후기·비용 투명화 시스템이 부재함
```

### (3) 해결 방향 (How)

- 우리는 이 문제를 어떻게 해결하고자 하는가?

```
•	플래너↔예비부부가 정보를 공유하는 실시간 협업 플랫폼 구축
•	AI 챗봇으로 웨딩 관련 질문 즉시 응답 및 맞춤 업체·일정 추천 제공
•	비용 명세를 항목별로 투명하게 공개하고, 일정·잔금 알림을 자동화하여 편의성 극대화
```

## 2 사용자 페르소나 & 유저맵 (User Persona & User Journey Map)

### **📌 사용자 페르소나**

**페르소나 1 — 예비신부 김지수 (29세, 직장인)**

- 결혼 준비 초기 단계, 스튜디오드레스·메이크업·웨딩홀 동시 알아보는 중
- 고충: 플래너 연락이 느리고, 비용 항목이 불분명해 예산 초과 불안감이 큼
- 목표: 진행 상황을 한눈에 파악하고, AI에게 언제든 질문할 수 있는 환경 필요
- 니즈: D-day 카운터타임라인·비용 명세를 실시간으로 확인, 챗봇으로 즉시 궁금증 해소

**페르소나 2 — 웨딩플래너 박지현 (32세, 경력 8년)**

- 동시에 12쌍의 커플을 담당, 일정비용 관리에 소요되는 행정 시간이 전체 업무의 40%
- 고충: 커플마다 카카오톡 채널이 달라 메시지 누락중복 응답이 잦음
- 목표: 담당 커플 현황을 한 대시보드에서 파악, 반복 행정 업무 최소화
- 니즈: 고객 목록잔금 마감·이번 주 할일을 한 화면에서 관리, 업체 카드 바로 공유

### **📌 사용자 유저맵 (User Journey Map)**

| **단계** | **예비부부 (커플)** | **웨딩플래너** |
| --- | --- | --- |
| **1. 문제 인식** | 비용 구조 불투명, 일정 파악 어려움 | 담당 커플 다수로 관리 복잡, 행정 과부하 |
| **2. 서비스 접근** | 플래너 초대코드로 앱 가입 및 연결 | 대시보드에서 신규 고객 등록·코드 발급 |
| **3. 핵심 사용** | 타임라인·비용·AI 챗봇으로 준비 진행 | 일정·비용 입력 → 커플에게 실시간 반영 |
| **4. 문제 해결** | 잔금 알림·업체 추천으로 의사결정 간편화 | 할 일 자동 생성으로 누락 없이 업무 처리 |
| **5. 재사용** | 결혼 완료 후 후기 작성, 지인 추천 | 신규 커플 추가 등록으로 지속 사용 |

### 📌 통계 기반 인사이트 (적용 배경)

- 국내 결혼 건수 연간 약 19만 쌍, 평균 웨딩 비용 3,200만 원 (통계청, 2023)
- 결혼 준비 스트레스 1위 — '비용 및 업체 관리 복잡성' (결혼정보회사 듀오 설문, 2023, n=1,200)
- 웨딩 관련 앱 사용 경험 없는 예비부부 68% — 전용 협업 도구 수요 입증 (자체 추정)
- AI 챗봇 활용 시 고객 응대 시간 평균 40% 단축 (Salesforce, State of Service Report, 2023)

## 5. 유사 서비스 비교 분석 **(Benchmarking)**

### 5.1 서비스 비교표

> ※ 3개 내외, 깊이보다 “차이점” 중심
> 

| 서비스명 | 주요 기능 | 강점 | 약점 |
| --- | --- | --- | --- |
| **웨딩북** | 업체 탐색, 후기, 견적 비교 | 업체 DB 풍부, 후기 많음 | 플래너↔커플 협업 기능 없음, AI 없음 |
| **허니문리스트** | 체크리스트, 예산 관리 | 준비 단계 안내 명확 | 플래너 연동 없음, 실시간 공유 불가 |
| **카카오톡+스프레드시트** | 메시지 소통, 비용 기록 | 범용성 높음, 익숙한 UI | 정보 유실 잦음, 자동화·AI 전무 |

### 5.2 **차별화 전략**

> ※ 비용 구조는 실제 금액이 아니라 무료/광고 없음/오픈소스 활용 등 
서비스 운영 관점에서 간단히 작성
> 

| 항목 | 유사 서비스 대비 차별점 |
| --- | --- |
| 사용 환경 : 언제, 누가 쓰는가 | 결혼 준비 전 기간(3~12개월), 플래너와 커플이 동일 플랫폼에서 실시간 협업 |
| 기능 차별화 : 무엇이 다른가 | 일정·비용·챗봇·업체를 하나로 통합 — 기존 서비스는 파편화된 기능 각개 제공 |
| 비용 구조 : 왜 가능한가 | 오픈소스 LLM API 활용, 광고 없는 클린 서비스로 MVP 비용 최소화 |
| 기술적 우위 : 훈련 연계성 | 웨딩 도메인 특화 프롬프트 설계, 플래너↔커플 실시간 데이터 동기화 |
| UX/UI : 사용자 관점 | Mongle 디자인 시스템 기반 — 타임라인·비용 명세 직관적 UI |
